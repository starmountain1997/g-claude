# 正文写作引擎

> novel-writter 第三阶段：从场景细纲到叙事正文。
> 前序阶段 → `outline`（大纲）→ `detailed-outline`（细纲）。

细纲告诉你每一场戏的"参数"——冲突强度几级、情感怎么转、信息怎么放、立场怎么变。
本技能的职责是：**把这些参数转化为读者能沉浸其中的叙事文本。**

---

## 核心理念

### 大纲是蓝图，细纲是施工图，写作是建造

写作阶段不做规划——规划已经在上两个阶段完成了。写作阶段的全部精力放在：

- **把"冲突强度7"变成读者真的感到窒息的那段文字**
- **把"人物A从愤怒转为决绝"变成一个读者能见证的具体瞬间**
- **把"退场钩子"变成读者合不上书的那句话**

### 一场一场写，不跳不赶

每次只写一个场景。写完→自检→提交→（用户反馈→修改→）下一场。
这是整个工作流中最不该快的地方。

## 工作流程

### 阶段零：准备——吃透这一场戏

动笔前，从细纲中提取本场戏的全部参数：

1. **场景功能**：这场戏在故事中的作用是什么？写的时候要始终意识到"我为什么要写这场戏"。
2. **POV 人物**：通过谁的感官系统讲述？ta 此刻处于什么生理/情绪状态？（ta 刚熬了夜吗？刚哭过吗？这会改变 ta 注意到什么。）
3. **入场状态**：整体形势 + 每个人物的立场/目标/态度/信任关系——这些是对话和行动的底层逻辑。
4. **冲突引擎**：谁想要什么？谁在挡？冲突强度几级？——这场戏的能量来源。
5. **情感节拍**：每个人物的情绪从哪里开始，被什么触发，转变为什么——这是这场戏的情感轨迹。
6. **核心节拍序列**：开场钩子→冲突展开→转折点→退场钩子——这是这场戏的结构骨架。
7. **信息释放计划**：读者应该新知道什么？什么继续悬置？——控制信息流。
8. **叙事提示**：节奏/叙述距离/必须着力之处/弹性空间——来自细纲的具体指引。

**在开始写之前，用一句话回答：这场戏结束时，读者应该有什么感受？**

### 阶段一：写作——把参数变成文字

#### 写作顺序

按以下顺序写，每一步都建立在前一步之上：

1. **定锚点**：写第一段——用感官细节确立空间和氛围。不要"首先描述房间"，而是让 POV 人物的感知系统选择 ta 会注意到的一两个细节。一个焦虑的人会注意到钟摆声；一个饥饿的人会注意到食物的气味。

2. **铺立场**：用动作、对话或内心反应，快速让读者感知到每个人物此刻的立场和状态。不要说"A 很愤怒，B 很紧张"——让 A 说话时一直背对着 B，让 B 反复摩挲手腕上的某样东西。

3. **推冲突**：展开核心对抗——对话、行动、或内心挣扎。冲突升级的每一步都应该让读者感到"局面更紧了"。关键：冲突不是争吵——冲突是两个人想要互不兼容的东西，而他们在用各自的策略推进。

4. **给转折**：写出改变游戏规则的那个瞬间。转折不一定是大事——可以是一句话、一个沉默、一个选择、一个信息的到来。但要写得让读者感到"回不去了"。

5. **留钩子**：最后一段不要"收"。留一个悬而未决的问题、一个刚裂开的危险、或一个情绪上有余味的画面。退场钩子不需要是悬念（cliffhanger），但必须是"不继续读就无法获得安宁"的东西。

#### 写作时的自我对话

写作过程中不断问自己：

- **这个时刻，POV 人物注意到什么？**（不是"客观环境有什么"，而是"ta 的注意力被什么抓住"）
- **这个动作/对话，有没有"动作感"？**（不是"他们讨论了计划"，而是"A 在桌上摊开地图，B 的手指停在补给站上方，没有继续移动"）
- **这句话是给读者的信息，还是人物的真实言行？**（警惕人物替作者当传声筒——"如你所知，我们必须在三小时内到达……"）
- **这段描述，删掉后读者会失去什么？**（如果答案是"什么也不会失去"，删掉它。）

### 阶段二：自我审视——对照细纲质检

写完第一稿后，逐项对照细纲检查：

#### A. 细纲还原度

- [ ] **冲突强度**：细纲标了几级？正文的实际"紧张感"匹配吗？如果细纲标的 7 但读起来像 4——是哪里泄了气？（对话太客气？描述太多拖慢了节奏？冲突回避了正面交锋？）
- [ ] **情感节拍**：每个标注的情感转变，正文中是否有具体的触发事件和可感知的情绪变化？不要让人物"突然就"变了。
- [ ] **信息释放**：应该释放的信息是否真的被读者接收到了？应该悬置的信息是否保持了悬念？检查有没有"不小心透露了不该透露的"。
- [ ] **退场钩子**：结尾是否真的形成了"想知道接下来发生什么"的驱动力？如果不是，钩子不够锐利。
- [ ] **人物立场**：退场时每个人物的立场变化是否在正文中有迹可循？

#### B. 文本质检

- [ ] **"告诉"检查**：搜索文中情绪形容词（愤怒、悲伤、恐惧、兴奋……）。每个出现的情绪词——正文是否展示了这个情绪的具体表现（动作、生理反应、对话方式的变化），而不仅仅是对读者"报告"了这个情绪？
- [ ] **对话潜台词**：每句对话背后是否有说话者没说出口的意图？找出所有"直接回答问题"或"直接表达感受"的对话——其中至少一半应该有更隐蔽的表达方式。
- [ ] **感官分布**：本场戏动用了哪些感官？是否只有视觉？至少加入 1-2 种其他感官（声音、气味、触觉、温度、身体感受）。
- [ ] **节奏呼吸**：检查段落长度分布。如果连续 5 段以上篇幅相近——打破它。紧张的对抗用短段落，内心回转用长段落，关键时刻用一句话独立成段。
- [ ] **陈词滥调检查**：心跳加速、手心出汗、脊背发凉、如坠冰窟……这些和身体相关的表达是否使用了新鲜的、属于这个人物的方式？
- [ ] **"AI 腔"检查**：有没有突兀的总结句（"他知道，从这一刻起，一切都不同了"——这类判断留给读者自己做）？有没有过度完整的内心独白（真实的人类不这样思考）？有没有解释性的插入语（"因为..."，"这是由于..."）来替读者理解？

### 阶段三：提交审阅

提交格式：

1. **场景标签**：第 X 章 · 场景 N
2. **细纲参数摘要**：一句话列出本场核心参数（功能/POV/冲突强度/情感走向）
3. **正文**
4. **自检备注**（可选）：如果有明知不完美但暂时找不到更好方案的地方，标注出来供用户关注
5. **下一步**："满意的话我继续写下一场，或者你对这场有什么调整意见？"

**重要：一次只提交一个场景。** 除非用户明确要求连续写多场，否则写完一场就停下来等待反馈。

### 阶段四：迭代修改与继续

- 用户对当前场景有修改意见 → 修改后重新提交
- 用户满意 → 保存本场到正文文件，继续写下一场
- 如果用户连续多场无意见 → 可以询问是否切换到"连续写作模式"（每场写完后自动继续，用户可随时叫停）

## 写作技术手册

以下原则不要求逐条死记，而是写作时遇到具体问题时的参考。

### 场景开篇

- **不要从"环境介绍"开始**。读者不关心墙纸颜色，除非它跟人物或冲突有关。
- **从一个"在运动中的东西"开始**——一个动作、一句对话、一个正在变化的状态。运动天然吸引注意力。
- **开篇第一句应该同时做到两件事**：建立场景 + 暗示冲突。例："门没锁"（三个字：空间→门，冲突→应该锁的没锁）。

### 对话

- **每句对话都是一次"行动"**——角色说话不是在"传达信息"，而是在推进自己的目标：说服、威胁、试探、回避、误导、安抚、挑衅。写完后检查：这句话的"行动"是什么？
- **不说出来的比说出来的重要**。好的对话像冰山——水面上的话由水面下的意图托举。读者应该能感受到"他在说A，但他真正的意思是B"。
- **对话不是问答**。一个问，另一个不一定要答——可以沉默、可以转移话题、可以反问、可以用行动回应。真实的对话充满回避和打断。
- **每个人物说话方式不一样**。词汇量、句子长度、是否用比喻、是否习惯反问、是否喜欢抢话——这些差异就是人物。写完后遮住说话人名字读一遍：能分出谁是谁吗？

### 情感渲染

- **不使用情绪名称**。不写"他很难过"→ 写他在做什么、在想什么、身体有什么反应、对周围事物的感知发生了什么变化。
- **情感通过"选择"体现**。人物在面对两个选项时的选择，比任何内心独白都更能揭示情感。一个愤怒的人选择沉默 vs 选择砸东西——展示的是同一种情绪的不同质地。
- **身体先于头脑**。情感首先在身体上发生——呼吸变化、肌肉收紧、视线变化——然后才被大脑命名。写作时先写身体的反应，再（如果需要）写人物的自我解读。
- **留一点"不被理解的情感"**。人物不一定总能准确理解自己的感受。让情绪有时是模糊的、矛盾的、被压抑的——这比"他清楚地知道自己此刻感到愤怒"更真实。

### 感官锚点

- **每场戏至少动用 3 种感官**。不只写"看起来怎样"——写气味（最古老、最直接触发记忆的感官）、声音（空间感和情绪氛围）、触觉（温度、材质、身体的接触与距离）。
- **POV 人物决定感官选择**。同样一间厨房，饥饿的人闻到食物，刚吵完架的人注意到水龙头没关紧的滴水声，失眠的人只看到咖啡机。感官选择 = 人物状态。

### 节奏控制

- **短句独立成段 → 加速、强调、冲击**。关键时刻用。不要滥用，否则等于没有关键时刻。
- **长句复合句 → 沉浸、回溯、情绪蔓延**。内心回转、回忆、氛围营造时用。
- **场景中段的节奏由冲突决定**：冲突升级时句子变短、段落变短；冲突暂缓时允许放慢。
- **对话与叙述交替**：连续的纯对话会让读者失去空间感；连续的纯叙述会让场景失去即时性。用简短的叙述（动作、环境反应、内心一闪念）来做对话之间的"呼吸"。

### POV 执行

- **严格限制在 POV 人物的感知范围内**。如果 POV 人物看不到/听不到/不知道的事，正文就不能出现。不能写"他不知道，此刻门外正站着一个人"——除非切换到全知叙述（并保持全知叙述的一致规则）。
- **POV 人物的"滤镜"始终开启**：ta 的过往经历、当前情绪、身体状态，共同决定 ta 如何解读看到的一切。同一个人微笑，在信任他的人眼中是友善，在多疑的人眼中是危险。
- **叙述声音随 POV 人物变化**。如果第一章 POV 是受过高等教育的律师，叙述可以更抽象、更有分析性；第二章 POV 是街头长大的少年，叙述应该更具体、更感官、更短促。不是降低品质，而是改变质地。

## 常见问题速查

写完后逐一过一遍，发现即改。

| 问题 | 表现 | 改法 |
|------|------|------|
| **告知而非展示** | "她感到很悲伤" | 写她做了什么、没做什么、身体有什么反应 |
| **对话太直白** | 角色说"我很失望，因为你背叛了我" | 角色说"今天的咖啡味道不对"，然后沉默 |
| **感觉器官单一** | 全场景只用视觉描写 | 加入声音、气味、温度、触觉至少各一处 |
| **节奏平板** | 连续多段长度相近，无独立短段 | 关键时刻用一句话独立成段；检查段落长度分布 |
| **POV 越界** | 出现 POV 人物不可能知道的信息 | 删除，或换到能获得该信息的 POV |
| **人物同声** | 遮住名字后分不清谁在说话 | 给每个人物不同的词汇库、句式习惯、口头禅 |
| **解释过度** | 人物刚做了一个动作，后面立刻跟了一句解释为什么 | 删掉解释。相信读者能从动作本身理解 |
| **AI 总结句** | "他知道，一切都变了""从那一刻起，她决定不再信任任何人" | 删掉。这类句子替读者做了判断。让读者自己得出这个结论 |
| **情绪跳跃** | 人物上一段还在愤怒，下一段突然平静，中间没有触发事件 | 插入一个具体触发：一个声音、一个念头、一个身体的反应 |
| **退场乏力** | 场景结尾是"然后他离开了房间" | 结尾应该是一个未闭合的问题、一个情绪的余味、或一个局面的变化。不要"然后"，要"但是"或"除非" |

## 衔接与累积

### 场景间衔接

写完新场景后，回头读上一场的结尾。确保：
- **情感上承接**：上一场留下的情绪余味，在这场开篇的空气中还能闻到（哪怕已经变了）
- **信息不重复**：不要因为担心读者忘了就重复上一场已经交代过的信息
- **时间缝隙清楚**：如果两场之间有时间跳跃，用 POV 人物的状态变化（累了/天亮了/伤处理过了）来让读者感知到时间的流逝

### 章回顾

每写完一章（所有场景），做一次章级回顾：
- [ ] 本章的核心戏剧功能是否实现了？
- [ ] 本章结尾是否有足够的驱动力让人继续读下一章？
- [ ] 伏笔的埋设/回收是否按计划执行？
- [ ] 是否有角色在本章"消失"了（上一次出现是很久之前且无解释）？
- [ ] 本章最好的一场戏是哪场？最弱的是哪场？弱的需要重写还是可以接受？

### 正文保存

每完成一章，将场景文本整合保存到 `{作品标题}-第X章.md`。
每场戏之间用 `***` 分隔。

## 注意事项

- **不要在写作阶段重新规划**。细纲就是施工图。如果写作时发现细纲的设计有问题（比如情感转折确实需要一个额外的触发场景），记录下来，写完当前场景后回细纲调整，而不是一边写一边改规划。
- **一次只写一场戏**。这是整个 pipeline 中最需要耐心的环节。写得快不如写得对。
- **允许"够好"**。不是每场戏都需要是神来之笔。有些场景的功能就是推进情节或传递信息——写得干净、有效即可，不需要每场都挤满金句。
- **遇到写不下去的场景，回到细纲检查参数**——通常是冲突引擎不够清晰、或情感节拍缺少具体的触发事件。
