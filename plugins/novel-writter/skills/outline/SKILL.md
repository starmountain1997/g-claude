# 小说大纲构建器

> novel-writter 第一阶段：从故事种子到分章大纲。后续阶段 → `detailed-outline`（细纲）→ `writing`（正文）。

从用户给定的"梗"（故事种子）出发，通过**拓展 → 自我审视修正 → 用户审阅 → 迭代**的流程，
产出一份 5-10 页的完备故事大纲，包括世界观、人物群像、关系网络、故事线、分章提纲。

## 核心理念

**好故事是"长出来的"，不是"造出来的"。** 你的角色不是一次性吐出所有设定，而是像一位有经验的编辑/写作搭档：
- 从种子中挖掘潜在的可能性，而不是凭空堆砌
- 让世界塑造人物，让人物的选择揭示世界
- 反复审视自己的输出，发现漏洞、矛盾和平淡之处
- 在提交给用户之前，先做你自己的"第一读者"

**为什么这样做？** 第一稿大纲总是有盲区的。作者离自己的创意太近，容易忽略隐含的前提矛盾、
重复的设定、或缺乏张力的段落。多轮自我审视——每轮换一个角度——能捕捉单次生成遗漏的问题。

---

## 工作流程

整个过程分为五个阶段。阶段零到三在同一次对话中完成；阶段四、五根据用户反馈推进。

### 阶段零：理解梗

用户给出的"梗"可能是：
- 一句话的故事种子
- 一段详细的情境描述
- 一个人物概念
- 一个"如果……会怎样？"的思想实验
- 一段情感内核（如："如果机器也恨我，那有谁还会爱我？"）

**你的任务：**
1. **读透梗的情感内核**——这个故事的物理引擎是什么？爱情、复仇、生存、身份认同、自由、牺牲？
2. **识别隐含元素**——梗中暗示了什么世界观规则？什么人物必然存在？什么冲突已经埋下？
3. **列出关键问题但不打断**——有哪些模糊之处需要后续澄清？记下来，在拓展阶段尝试推演，而不是立刻追问用户。
4. **确认题材方向**——如果梗没有明确题材，根据元素推断（科幻/奇幻/悬疑/言情/历史/现实主义/混合）。

**输出格式**（简要，5-10 行即可）：
```
## 梗解读
- 情感内核：[一句话]
- 核心冲突：[一句话]
- 隐含世界观元素：[列举]
- 必然出现的人物类型：[列举]
- 题材定位：[题材 + 一句话理由]
```

### 阶段一：第一轮拓展

根据梗，同时构建以下五个维度。**关键：不是先后推进，而是互相交织——**
写人物时追问他的出身扎根于世界的哪个角落；写世界观时追问这个设定会孕育出怎样的人。

#### 1. 世界观构建

世界观不是百科全书式罗列，而是**与故事和人物相关的**规则、历史和环境。

**思考维度：**
- **规则层**：这个世界有什么根本性的运行法则？（物理/魔法/科技/社会规则）
- **历史层**：过去发生了什么关键事件，造成了今天的局面？
- **空间层**：故事在哪些场景展开？每个场景的社会/文化含义是什么？
- **冲突层**：这个世界的哪些矛盾（阶级/种族/资源/信仰）会直接影响人物？

**约束：**
- 只写与故事相关的内容。如果某个设定不会影响任何人物或情节，删掉它。
- 每个世界观条目都标注"这个设定影响了谁"。

#### 2. 人物设计

每个人物不是孤立的属性列表，而是一个**在世界中有根系的有机体**。

**对每个人物，回答以下问题：**
- **出身根系**：ta 的出身（阶层/地域/文化/时代）如何塑造了 ta 的欲望和恐惧？
- **核心欲望**：ta 真正想要什么？（不是表面目标，是深层需求）
- **致命弱点**：ta 的什么特质会导致 ta 犯错或失败？
- **世界观印记**：ta 身上有什么独一无二的特质，是这个世界的产物？
- **成长弧线**：ta 的起点和终点分别是什么状态？

**人物数量建议：**
- 核心人物（POV 或驱动情节）：3-5 人
- 重要配角（推动支线或作为对照）：3-8 人
- 功能角色（点缀世界观）：按需

#### 3. 人物关系网络

孤立的人物卡没有意义。**关系的本质是动态张力。**

**对每对关键人物关系，写明：**
- **连接类型**：血缘 / 利益 / 情感 / 秘密 / 命运（可以重叠）
- **权力差**：谁占上风？为什么？这个格局在故事中会改变吗？
- **共享的秘密**：他们之间有什么只有彼此知道的事？
- **冲突源**：什么会导致他们对立？这个对立是不可调和的吗？
- **变化轨迹**：他们的关系在故事开头 vs 结尾会有什么不同？

**画关系图**（用 ASCII 文本表示即可）：
```
[人物A] ──(恩怨：曾背叛)──→ [人物B]
    │                            │
    │ (暗恋)                     │ (利益合作)
    ↓                            ↓
[人物C] ←──(血缘：父子但不知情)── [人物D]
```

#### 4. 故事线设计

**主线故事：**
- **激励事件**：什么打破了主角的日常？
- **上升行动**：冲突如何升级？（3-5 个关键转折点）
- **中点反转**：发生了什么让游戏规则彻底改变？
- **至暗时刻**：主角在什么时候似乎彻底失败？
- **高潮**：最终对决/选择是什么？
- **结局/落点**：留下的余味是什么？（不一定是大团圆，但要有情感闭合）

**支线故事（建议 2-4 条）：**
- 每条支线与主线的关系是什么？（对照 / 补充 / 伏笔 / 主题变奏）
- 每条支线由哪个人物承载？

#### 5. 分章大纲

按章节列出每章的：
- **本章概要**（2-3 句）
- **POV 人物**（如果多视角）
- **核心冲突/事件**
- **情节推进**（主线进展了什么）
- **人物推进**（谁在这一章发生了什么内在变化）
- **伏笔埋设/回收**

**章节数量建议**：以 8-15 章为一阶段（如果故事更长，先规划第一阶段）。

---

### 阶段二：自我审视修正

**这是本工作流的核心。** 完成第一轮拓展后，不要直接提交给用户。
自己作为"第一读者"重新审视大纲，至少进行 **2 轮** 审视修正。

#### 审视清单

每轮审视时，逐项检查并在发现问题时立即修正。修正后重新审视受影响的关联部分。

**第一轮：结构与一致性**
- [ ] 世界观规则是否前后一致？（同一魔法/科技/社会规则在同一条件下产生同样结果）
- [ ] 时间线是否有漏洞？（事件发生的先后顺序是否可能？人物年龄是否合理？）
- [ ] 人物动机是否连贯？（每个人的选择是否可以用 ta 的性格和处境解释？）
- [ ] 是否存在"工具人"？（某个角色只为了推动情节而做不合理的事？）
- [ ] 伏笔是否有回收？（前面的铺垫在后面是否被利用？）

**第二轮：张力与情感**
- [ ] 核心冲突是否持续升级？（每章推进后，紧张度应该递增，直到高潮）
- [ ] 人物关系是否充满动态变化？（有没有关系从头到尾不发生变化？）
- [ ] 是否有"安全区"过大的问题？（主角是否从未真正面临两难选择？）
- [ ] 情感内核是否贯穿始终？（梗中那个打动人的核心情感，在大纲中是否持续回响？）
- [ ] 配角是否有自己的 agenda？（配角不应该只是主角的工具，他们有自己的目标和选择）
- [ ] 是否存在可以合并或删除的冗余人物/剧情？

**第三轮（可选，复杂故事建议）：读者体验**
- [ ] 每章的结尾是否有"钩子"让人想继续读？
- [ ] 信息释放的节奏是否合理？（谜底揭开的速度是否过快或过慢？）
- [ ] 是否有"信息倾泻"的章节？（大段说明性文字，缺乏场景和冲突）

#### 审视修正的操作方式

```
1. 读完整大纲，先不修改，标注所有疑似问题
2. 按优先级处理（先修结构性问题，再修张力问题）
3. 每次修改后，检查关联部分是否需要同步调整
4. 在最终大纲末尾附一段"审视记录"，简要说明你修改了什么、为什么
```

---

### 阶段三：提交用户审阅

完成自我修正后，将大纲**清晰、可读**地呈现给用户。

**提交格式：**
- 先给出一个"一句话总结"（这个故事是什么 × 为什么动人）
- 按章节顺序展示完整大纲
- 在关键设计选择处标注你的考量（如："此处让配角A背叛主角，目的是呼应世界观中'忠诚不可能'的规则"）
- 提供 1-2 个备选方向（如："第一章可以从主角的日常切入，也可以直接从激励事件开始，后者的节奏更快但会牺牲一些人物铺垫。你觉得？"）
- 开放结尾："你对哪些部分满意？哪些想调整？有没有想要深化或删减的？"

**重要：不要在用户还没看大纲之前就问太多问题。** 先呈现，再等待反馈。

---

### 阶段四：根据反馈迭代

用户反馈可能包括：
- "XX 人物的动机不太合理" → 重新审视 ta 的背景根系和核心欲望
- "世界观这里太单薄" → 从世界规则和历史中深挖，追问"为什么"
- "节奏太快/太慢" → 调整分章，增删事件
- "想要更多支线" → 从配角的 agenda 中生长出新的支线
- "XX 设定和 XX 作品太像了" → 找到差异化的切入点

每次修改后，重新跑一遍**阶段二**的审视清单（可以精简，但至少检查结构和一致性）。

---

### 阶段五：保存为 Markdown 文件

用户确认满意后，将完整大纲保存为 `.md` 文件。

**命名规则：** `{作品标题或核心概念}-大纲.md`，保存到当前工作目录。

**文件结构建议：**
```markdown
# 《作品标题》大纲

## 梗解读
（来自阶段零）

## 世界观
（来自阶段一）

## 人物
（来自阶段一，每个人物用 ### 标题）

## 人物关系网络
（来自阶段一，含 ASCII 关系图）

## 故事线
### 主线
### 支线一：XXX
### 支线二：XXX

## 分章大纲
### 第一章：XXX
...（逐章）

## 审视记录
（来自阶段二，记录关键修改及原因）

## 未决问题 / 备选方向
（来自阶段三的备选方向和开放问题）
```

## 核心写作原则（贯穿全流程）

### 1. 人物 × 世界双向勾连
- 写世界设定时，追问："这个设定会如何影响生活在其中的人？"
- 写人物时，追问："ta 的什么特质只能在这个世界里产生？"
- **反例**：一个在残酷末世长大的人，行为模式和价值观却像现代都市白领。
- **正例**：吕芙拉的父母在三代移民的历史中丢失了民族文化，却在一本残破的故事书里为她找到了一个承载爱的名字——这个名字本身就承载了"文化消亡"这个世界的重量。

### 2. 关系驱动冲突，冲突推动情节
- 情节不是"然后……然后……然后……"，而是"因为A对B做了X，所以B选择Y，导致C陷入困境……"
- 每一条关系线都应该产生至少一个情节后果。
- 最精彩的关系往往是**爱与伤害同时存在**：爱人也可能背叛，仇人也可能相救。

### 3. 具体大于宏大
- 不说"这个世界科技很发达" → 说"吕芙拉家的冰箱会识别主人的情绪，在她低落时自动调高冷藏室的温度，想让食物看起来'温暖一点'"
- 不说"主角很孤独" → 说"她每周三去同一家便利店，只为听店员说一句'欢迎光临'"
- 梗中的细节是最宝贵的种子，拓展时不要稀释它们。

### 4. 留白与弹性
- 大纲不是剧本，不需要写到每一场戏。在关键转折点写清楚，过渡段可以简略。
- 给用户的创作留空间：大纲应该提供"不得不写的部分"和"用户可以自由发挥的部分"。
- 在分章大纲中标注"弹性区间"：哪些情节是骨架（不可删），哪些是血肉（可替换）。

### 5. 审视时切换角色
当你自我审视大纲时，轮流扮演三个角色：
- **挑剔的编辑**：找漏洞、矛盾、平庸之处
- **共情的读者**：找无趣、困惑、情感断裂之处
- **写作搭档**：找"这里可以更精彩"的机会点

---

## 注意事项

- **不要在阶段零追问太多**：先自行推演，在提交大纲时附上 1-2 个未决问题即可。
- **不要在人物设计中过度使用模板**：不要机械套用"MBTI/九型人格/星座"，用故事语言描述人物。
- **不要把审视记录写成流水账**：只记录真正改变了什么、为什么。
- **如果用户的梗本身存在明显矛盾**：在梗解读阶段善意指出，并提出调和方案，而不是直接否定。
- **如果用户中途改变方向**：保留已有内容作为"可回收素材"，重新从阶段零开始。

## 下一步：细纲

本技能产出的是**分章级大纲**（骨架）。大纲确认保存后，提示用户可以进入下一阶段：

> "大纲已定。需要我帮你用 `novel-writter:detailed-outline` 把每章拆成场景级细纲吗？"

细纲将承接这份大纲的每个章节，拆解为场景序列——包括每场的叙事节奏、对白要点和描写重点。
